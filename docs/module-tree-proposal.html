Dendritic module tree — proposal

The old flake.old.nix + modules.old.d/ (98 nix files, 3 hosts) restructured into the dendritic pattern, organized into branches/groups — terminals, shell, dev, wm, desktop, hardware, network, system, ai, … — each a set of single-purpose modules (git, clojure, input/xkb, …) that merge into flake.modules.<class>.<bucket> and are auto-loaded by import-tree.

All six decisions resolved. Data-driven — edit the RADIAL / TREE objects to iterate.

    Decisions (all resolved)

    1. GUI apps → home-manager. [RESOLVED] media (discord/spotify), office, zathura, browsers are home-manager modules in the desktop branch; daemons/services/hardware stay nixos.
    2. WM helper config → wrappers only. [RESOLVED] rofi/waybar/dunst/swaylock/swayidle configured once in wrappers/*; HM programs.* copies dropped.
    3. Niri → deferred. [RESOLVED] Ported later under pkgs/niri/. For now euler = i3, laplace = sway.
    4. Desktop → layered core/wayland/x11. [RESOLVED] core ← {wayland|x11} ← WM ← host via bucket-to-bucket imports.
    5. Secrets → central wiring + per-feature decls. [RESOLVED] flake/secrets.nix wires sops-nix; each feature declares its own secret.
    6. Options → bare per-domain, import = enable. [RESOLVED] No me.*/my.*; git.userEmail, one username, flakePath. Importing a module enables it.
