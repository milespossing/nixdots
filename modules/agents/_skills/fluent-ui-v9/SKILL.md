# Fluent UI v9 — Read-On-Demand

This skill is a **routing table**, not a docs mirror. For any v9
question, fetch the authoritative file directly from
`microsoft/fluentui` on GitHub.

## Prefer the FluentAgent MCP when it's available

If the session has the `fluent-agent` MCP server connected (URL
`https://chat.fluentui.dev/mcp/server/`, OAuth via Microsoft corp
AAD), **try its tools first** for component, token, and example
lookups. It's Microsoft's official agent surface on the same domain
as the v9 docs, so its answers track upstream automatically.

Fall through to the GitHub paths below when:

- The MCP isn't connected (auth not yet completed, non-Microsoft host).
- You need the **raw source file** verbatim (e.g. to read an exact
  `*.api.md` type signature, or to copy a `*.stories.tsx` example
  unchanged).
- The MCP answer is missing a field you need (slots list, exact
  package name, branch-tip behavior).

## Procedure

### 1. Find the package name

v9 component packages live under
`packages/react-components/`. Naming is **`react-<kebab-name>`**:

| Component         | Package                       |
| ----------------- | ----------------------------- |
| `Button`          | `react-button`                |
| `Dialog`          | `react-dialog`                |
| `Avatar`          | `react-avatar`                |
| `DataGrid`        | `react-data-grid-react-window`|
| `Accordion`       | `react-accordion`             |

If the package name isn't obvious, list them all (once, cache mentally
for the session):

```bash
gh api /repos/microsoft/fluentui/contents/packages/react-components \
  --jq '.[] | select(.type=="dir") | .name'
```

There are ~67 packages.

### 2. Fetch the right file for the question

Replace `<pkg>` with the package from step 1 and `<Name>` with the
PascalCase component name.

| You need…                       | Path                                                                      |
| ------------------------------- | ------------------------------------------------------------------------- |
| Props, slots, types (canonical) | `packages/react-components/<pkg>/library/etc/<pkg>.api.md`                |
| Long-form spec, variants        | `packages/react-components/<pkg>/library/docs/SPEC.md`                    |
| v8 → v9 migration               | `packages/react-components/<pkg>/library/docs/MIGRATION.md`               |
| One-line description            | `packages/react-components/<pkg>/stories/src/<Name>/<Name>Description.md` |
| Best practices (dos/don'ts)     | `packages/react-components/<pkg>/stories/src/<Name>/<Name>BestPractices.md` |
| Accessibility spec              | `packages/react-components/<pkg>/stories/src/<Name>/<Name>AccessibilitySpec.mdx` |
| Canonical usage examples (code) | `packages/react-components/<pkg>/stories/src/<Name>/<Name><Variant>.stories.tsx` |
| List of all example variants    | `packages/react-components/<pkg>/stories/src/<Name>` (directory listing)  |

Fetch with either:

```bash
# Authenticated, full repo APIs:
gh api -H "Accept: application/vnd.github.raw" \
  /repos/microsoft/fluentui/contents/<path>

# Anonymous, faster for plain reads:
# https://raw.githubusercontent.com/microsoft/fluentui/master/<path>
```

The `.api.md` is the **source of truth** for prop names, types, and
slots — it's auto-generated by API Extractor from the actual TS source
and updated every commit. When in doubt, this is the file to read.

### 3. Design tokens

Two layers, both in the same repo:

| You need…                            | Path                                                                       |
| ------------------------------------ | -------------------------------------------------------------------------- |
| Primitive color ramps (greys, brand) | `packages/tokens/src/global/colors.ts`                                     |
| Other primitives (typography, etc.)  | `packages/tokens/src/global/` (directory listing)                          |
| Semantic tokens (`tokens.colorXyz`)  | `packages/react-components/react-theme/library/src/global/` (listing) and `…/light/`, `…/dark/`, `…/highContrast/` for theme-specific values |

The semantic tokens in `@fluentui/react-theme` are what you actually
use in `makeStyles` (`tokens.colorNeutralForeground1` etc.). The
primitives in `@fluentui/tokens` are what those resolve to.

### 4. Rendered output (Storybook)

If you need to **see** a component rendered (not just read code), the
v9 Storybook is at `https://react.fluentui.dev/`. Story IDs follow
the pattern `components-<pkg-suffix>-<name>--<variant>`, e.g.
`components-button-button--default`. The full index is at
`https://react.fluentui.dev/index.json`.

To screenshot a story, use the [`browser-control`](../browser-control/SKILL.md)
skill and open
`https://react.fluentui.dev/iframe.html?id=<story-id>&viewMode=story`.

## Example: "What props does Dialog take?"

```bash
gh api -H "Accept: application/vnd.github.raw" \
  /repos/microsoft/fluentui/contents/packages/react-components/react-dialog/library/etc/react-dialog.api.md
```

Read the resulting markdown. Every `export type Dialog*Props` and
`export type Dialog*Slots` is listed with full TS signatures.

## Example: "Show me canonical usage of Avatar with status"

```bash
# 1. List variants:
gh api /repos/microsoft/fluentui/contents/packages/react-components/react-avatar/stories/src/Avatar \
  --jq '.[].name'
# 2. Fetch the one that looks right:
gh api -H "Accept: application/vnd.github.raw" \
  /repos/microsoft/fluentui/contents/packages/react-components/react-avatar/stories/src/Avatar/AvatarBadge.stories.tsx
```

## Notes

- Branch is `master`, not `main`.
- The `microsoft/fluentui` repo enforces a 90-day max lifetime on
  fine-grained PATs at the enterprise level. `gh` CLI tokens
  (classic / device-flow) are fine; the GitHub MCP server's typed
  endpoints may be blocked depending on its token. If `gh api` works
  and the MCP doesn't, use `gh api`.
- Do **not** cache fetched content into this repo. The whole point is
  that the upstream is the source of truth.
